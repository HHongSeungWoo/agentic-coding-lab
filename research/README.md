# Research Notes

This directory contains first-class research notes about agentic coding techniques and whether they should become implemented artifacts.

Research here is Markdown-first. The index tracks discovery and triage snapshots, while individual notes capture the reasoning needed to decide whether an idea should become an Agentic Coding Artifact.

## Repository discovery categories

- `skills-instructions`
- `mcp`
- `harness-eval`
- `agent-support-systems`

Repository research excludes agent applications and model clients as direct subjects. Frameworks are conditional: review only subsystem patterns that can improve agentic coding above another tool.

## Paper discovery topics

- `token-efficiency`
- `context-control`
- `memory`
- `subagents-multiagents`
- `tool-use`
- `ai-coding-workflow`
- `error-prevention`
- `domain-specific-coding`

Paper research includes recent and influential work when the method or evidence can inform a support system for AI coding agents.

## Workflow

1. Add repository and paper candidates to `index.md` with source snapshots.
2. Triage candidates as `in-scope`, `conditional`, or `out-of-scope`.
3. Deep-review accepted repository candidates from temporary clones outside this repository.
4. Deep-review accepted papers from accessible paper text, project pages, code, and benchmark sources.
5. Write notes from `templates/`.
6. Promote cross-cutting lessons into `synthesis/`.
